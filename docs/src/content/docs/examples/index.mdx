Examples show Caracal integrated into concrete applications. Use them after completing the introductory tutorials and before building your own production flow.

## Available examples

| Example | What it demonstrates |
| --- | --- |
| [Lynx Capital: Autonomous Payouts](/examples/lynx-capital/) | FastAPI + LangGraph agent swarm, Console setup, Control key, runtime profile, provider resources, Caracal SDK, Gateway, STS, Coordinator, and live run inspection. |
| Echo Upstream (`examples/echoUpstream`) | A zero-dependency local upstream that makes the first tutorial self-contained, so you can confirm a brokered Gateway call without hosting an API of your own. |
| Control Bootstrap (`examples/controlBootstrap`) | A non-interactive script that creates and tears down a demo zone, resource, and policy through the Control API, for repeatable per-laptop or CI setup. |
| Provider Preflight (`examples/providerPreflight`) | An automatable preflight that validates a resource-to-provider binding, OAuth token endpoint host, callback and upstream reachability, runtime-injection eligibility, and policy authorization before the first Gateway call. |

## How to read examples

1. Read the scenario and architecture first.
2. Start Caracal with the released runtime/Console path described by the example.
3. Use Console to create real zones, resources, provider configuration, and profiles.
4. Run the example tests before changing behavior.
5. Keep provider fixtures separate from Caracal product state.
