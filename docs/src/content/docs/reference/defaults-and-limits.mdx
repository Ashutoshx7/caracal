This page is the single source of truth for default values across Caracal. Every value here is either a documented default in the service configuration or a hard ceiling enforced by the implementation. Use it when sizing a deployment, tuning a policy, or troubleshooting unexpected expirations.

## Mandates and sessions

| Value | Default | Configurable via | Notes |
| --- | --- | --- | --- |
| Resource mandate TTL | 15 minutes | Policy template | Maximum lifetime of an STS-issued mandate for a resource call. |
| Session mandate TTL | 60 minutes | Policy template | Maximum lifetime of a session-scoped mandate. |
| Agent session TTL | 3600 seconds | Coordinator config | Coordinator-managed agent session lifetime. |
| Delegation edge TTL | Bounded by parent | Delegation request | A delegated edge cannot outlive its parent edge or session. |
| JTI replay window | Mandate TTL + 60s clock skew | `JTI_TTL_SECONDS` | How long the gateway remembers a JTI for replay detection. |

## Hard ceilings (not configurable)

- A single grant may reference up to 64 scopes.
- A single delegation chain may not exceed 8 hops.
- A single policy set may have up to 16 active versions retained.
- Audit `metadata_json` and `diagnostics_json` are truncated at 16 KB each.

These ceilings are enforced server-side and are visible in error responses as `limit_exceeded` with a `field` indicating which limit was hit.
