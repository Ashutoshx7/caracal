## Prerequisites

- Docker Engine 24+ with Compose v2, or Docker Desktop 4.x
- Git 2.x
- `curl` or `wget` (Linux/macOS)
- PowerShell (Windows)

## Manual install

Download the binary for your platform from the [latest release](https://github.com/Garudex-Labs/caracal/releases/latest). Verify the download against `SHA256SUMS`, place it on your `PATH`, and make it executable on Unix (`chmod +x caracal`).

| Platform | Binary name |
|---|---|
| Linux x64 | `caracal-linux-x64` |
| Linux arm64 | `caracal-linux-arm64` |
| macOS x64 | `caracal-darwin-x64` |
| macOS arm64 (Apple Silicon) | `caracal-darwin-arm64` |
| Windows x64 | `caracal-windows-x64.exe` |

## Environment variables

The CLI reads these variables. Set them in your shell or in a `.env` file in the project root.

| Variable | Default | Notes |
|---|---|---|
| `CARACAL_ADMIN_TOKEN` | — | Required for all admin operations. Printed by the installer. |
| `CARACAL_API_URL` | `http://localhost:3000` | |
| `CARACAL_COORDINATOR_URL` | `http://localhost:8080` | |
| `CARACAL_COORDINATOR_TOKEN` | — | Required for `agent` and `delegation` commands. |
| `CARACAL_ZONE_ID` | — | Or set `zone_id` in `caracal.toml`. |

## Start the runtime stack

```bash
# Start all services (STS, Gateway, Coordinator, Audit) via Docker Compose
caracal up

# Provision the local zone and write caracal.toml
caracal init

# Verify all services are healthy
caracal status

# Stop all services; add -v to remove persistent volumes
caracal down
```

## Validation

After `caracal status` reports all services healthy, the following endpoints are reachable:

| Endpoint | Expected response |
|---|---|
| `http://localhost:8080/health` | `{"status":"ok"}` |
| `http://localhost:8081/health` | `{"status":"ok"}` |
| `http://localhost:8082/healthz` | `{"status":"ok"}` |
