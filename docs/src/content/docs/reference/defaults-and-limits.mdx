Use this page when sizing a deployment, writing automation, or troubleshooting unexpected expirations. Values listed here are backed by current service configuration, terminal management interface behavior, or hard-coded protocol limits.

## Token exchange and authority

| Value | Default | Configurable via | Notes |
| --- | --- | --- | --- |
| Resource mandate TTL | 15 minutes | `ttl_seconds`, capped at 15 minutes | STS default and maximum for resource-bound exchanges. |
| Session mandate TTL cap | 60 minutes | `ttl_seconds`, capped at 60 minutes | Applies only when STS is allowed to issue a session mandate. |
| `caracal run` token request | 3600 seconds | Not configurable by terminal management interface flag | The terminal management interface requests the session-mandate cap before injecting configured credentials. |
Open the terminal management interface and follow the matching view.
| Subject-token validation leeway | 60 seconds | Not configurable | STS accepts issuer, audience, issued-at, and expiry clock skew within this leeway. |
| Zone signing key cache | 15 minutes | Not configurable | STS keeps decrypted zone signing keys in memory for this interval. |
| Issued-JTI registry TTL | Token TTL | Not configurable | STS stores minted JTIs in Redis until the corresponding token expires. |
| Gateway JTI replay marker | Time until token expiry | Not configurable | Gateway rejects duplicate JTI presentations within the token lifetime. |
| Delegated token TTL | Minimum of requested TTL, edge expiry, and `ttl_seconds` constraint | Delegation constraints | Delegated exchanges cannot outlive the active edge or an explicit delegation TTL constraint. |

## Hard ceilings (not configurable)

- `OPA_POLL_SECONDS` must be less than or equal to `300`.
- Gateway `PORT` must be `8081`; STS `PORT` must be `8080`; Audit `PORT` must be `9090`.
- Gateway `READ_TIMEOUT` must be greater than `STS_TIMEOUT + UPSTREAM_TIMEOUT`.
- `JTI_FAIL_OPEN` is forbidden when `CARACAL_MODE=rc` or `CARACAL_MODE=stable`.
- Delegation `max_depth`, `max_hops`, `ttl_seconds`, and `budget` constraints must be non-negative. If neither `max_depth` nor `max_hops` is set, delegation proof validation uses `1` hop.
