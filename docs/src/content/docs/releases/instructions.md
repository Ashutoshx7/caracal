# Releases

## Scope
- Only per-version release validation reports written by the post-release harness.

## Required
- One report per published version, named `v<semver>-validation.md`.
- Each report must be generated by `caracal/scripts/postRelease/aggregateReport.ts`.

## Forbidden
- Must not hand-edit reports after the harness commits them; rerun the workflow instead.
- Must not add narrative release notes here; those belong under `docs/src/content/docs/contributing/release.mdx`.
