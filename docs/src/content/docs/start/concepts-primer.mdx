## Overview

Caracal has nine primitives. Each one maps to a concrete thing in the runtime. Understanding all nine takes about five minutes; after that, every other doc page becomes self-explanatory.

The primitives form three layers:

- **Identity layer** — who is acting: [Principal](#principal), [Workspace](#workspace), [Zone](#zone)
- **Authority layer** — what they're authorized to do: [Policy](#policy), [Mandate](#mandate), [Intent](#intent), [Caveat](#caveat)
- **Record layer** — what happened: [Ledger](#ledger), [Delegation](#delegation)

---

## Principal

> The identity that requests authority.

A **Principal** is any authenticated identity: an application service, an AI agent, or a human user. Every request for a mandate must come from a principal. Principals belong to a [Workspace](#workspace).

Three types:
- `application` — a service or agent that authenticates with a signed JWT assertion or shared secret.
- `user` — a human, authenticated via the coordinator's identity binding flow.
- `delegate` — an agent that received authority from another principal via [Delegation](#delegation).

[Full reference →](/concepts/principal/)

---

## Policy

> Declarative rules that decide what a principal may do.

A **Policy** is a bundle of [OPA Rego](https://www.openpolicyagent.org/docs/latest/policy-language/) rules stored in a [Workspace](#workspace). The STS evaluates the policy on every mandate request. The policy returns: `permit`, `deny`, `step-up` (require more auth), or `constrain` (permit with attached [Caveats](#caveat)).

```rego
package caracal.permit
allow { input.principal.type == "application"; input.intent.action == "file:read" }
```

[Full reference →](/concepts/policy/)

---

## Intent

> The declared action a principal is about to take.

An **Intent** is the structured declaration embedded in a mandate request: an `action` verb (e.g., `file:read`), a `resource` target (e.g., `/data/reports/*.csv`), and optional `context`. The STS evaluates the intent against the policy. The gateway checks that the actual outbound request matches the declared intent.

[Full reference →](/concepts/intent/)

---

## Mandate

> The signed authority object that proves a principal may act.

A **Mandate** is a short-lived ES256-signed JWT the STS issues after a successful policy evaluation. It embeds the principal ID, the policy decision, and the declared intent. The gateway requires a valid mandate on every request. Mandates expire (default: 300 seconds) and can be revoked mid-session.

Think of a mandate as a signed, scoped, time-bounded permission slip — not a credential.

[Full reference →](/concepts/mandate/)

---

## Caveat

> A constraint that narrows a mandate.

A **Caveat** is a constraint embedded in a mandate by a policy `constrain` rule. Types: TTL reduction, resource pattern restriction, action allowlist, context requirement. Caveats only narrow authority — they cannot expand it. Multiple caveats on one mandate are all enforced; the most restrictive wins.

[Full reference →](/concepts/caveat/)

---

## Delegation

> Passing scoped authority to another principal.

**Delegation** is how an orchestrator agent gives a sub-agent the authority it needs for a specific task — and only that. The delegating principal issues a delegation token via the coordinator. The sub-agent presents that token to become a `delegate` principal. The delegate can never hold more authority than the delegator.

[Full reference →](/concepts/delegation/)

---

## Workspace

> The policy and principal boundary.

A **Workspace** groups together one policy bundle, the principals registered within it, and the ledger entries for decisions made in it. It is Caracal's isolation boundary: a principal from workspace A cannot receive mandates governed by workspace B's policy.

[Full reference →](/concepts/workspace/)

---

## Zone

> The deployment unit that owns signing keys.

A **Zone** owns the ES256 signing key pairs used to sign mandates and the zone encryption key (ZEK) used to vault provider credentials. Zones contain workspaces. One STS instance manages one zone. The zone's public keys are published via a JWKS endpoint; gateways cache them to verify mandate signatures.

[Full reference →](/concepts/zone/)

---

## Ledger

> The tamper-evident record of every authority decision.

The **Ledger** is an append-only log of every mandate issuance, denial, gateway verification, revocation, and delegation event. It is populated via a Redis stream and persisted by the audit service. The ledger is workspace-scoped and supports replay: re-running a past session against a policy.

[Full reference →](/concepts/ledger/)

---

## How they connect

```
Zone
 └── Workspace
      ├── Policy ──────────────────► evaluates ──► Intent
      ├── Principal ───────────────► produces ───► Mandate
      │    └── (via Delegation) ──► delegate principal
      └── Ledger ◄── records ◄──────────────────── Mandate
                                                      │
                                              constrained by Caveat
```

Every call goes through this loop: Principal declares Intent → Policy evaluates → STS issues Mandate → Gateway verifies Mandate → Ledger records the decision.
