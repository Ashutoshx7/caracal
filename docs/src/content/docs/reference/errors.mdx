Caracal packages and services use a shared machine-readable error shape.

```json
{
  "error": "invalid_token",
  "error_description": "bearer signature invalid",
  "requestId": "018f...",
  "details": {}
}
```

`details` appears only when a service or package provides structured context.

## Well-known codes

| Code | Meaning |
| --- | --- |
| `access_denied` | Request is authenticated but not authorized. |
| `invalid_token` | Token is missing, malformed, expired, invalid, replayed, or fails verification. |
| `resource_not_found` | Requested resource does not exist or is unavailable in the current scope. |
| `internal_error` | Service failed unexpectedly. |
| `policy_eval_failed` | Policy evaluation failed. |
| `provider_rate_limited` | Provider or provider coordination rate limit denied work. |
| `interaction_required` | Step-up challenge is required. |
| `sts_unavailable` | STS could not satisfy an exchange. |
| `credential_expired_not_renewable` | Credential is too close to expiry or cannot be renewed. |
| `payload_too_large` | Request body or token exceeded the configured limit. |
| `zone_invalid` | Zone claim or route zone is invalid. |
| `scope_insufficient` | Required scope is missing. |
| `agent_identity_required` | A verifier requires an agent identity. |
| `delegation_required` | A verifier requires delegated authority. |
| `chain_mismatch` | Delegation chain does not contain a required application/session. |
| `hop_count_exceeded` | Delegation hop count exceeds the allowed maximum. |
| `http_request_failed` | HTTP call failed before a valid response could be used. |
| `config_missing` | Runtime or service configuration is missing. |

Transport packages may map these codes into framework-specific HTTP responses.
