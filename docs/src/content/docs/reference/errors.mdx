## Error format

All Caracal services return errors as JSON:

```json
{
  "error": "mandate_expired",
  "error_description": "The mandate has expired. Request a new mandate via the coordinator."
}
```

The SDK surfaces these as typed exceptions:
- `CaracalError.MandateExpired`
- `CaracalError.PolicyDenied`
- `CaracalError.Revoked`
- `CaracalError.IntentMismatch`
- `CaracalError.ReplayDetected`
- `CaracalError.Unauthenticated`

## STS errors (`POST /oauth/2/token`)

| HTTP | Error code | Meaning | Remediation |
|---|---|---|---|
| 400 | `invalid_request` | Missing or malformed required parameter. | Check all required fields: `grant_type`, `subject_token`, `subject_token_type`, `resource`. |
| 400 | `invalid_client` | Application assertion signature verification failed. | Verify the assertion's `iss` matches `application_id`; check key rotation. |
| 400 | `invalid_grant` | Subject token is expired, revoked, or not recognized. | Request a new mandate from the coordinator. |
| 400 | `unsupported_grant_type` | `grant_type` is not `urn:ietf:params:oauth:grant-type:token-exchange`. | Fix the grant type in the client. |
| 401 | `unauthenticated` | Application credentials missing or not recognized. | Ensure `CARACAL_APP_ID` and `CARACAL_APP_SECRET` are set correctly. |
| 403 | `policy_denied` | OPA policy evaluated the intent as `deny`. | Review the workspace policy `caracal.deny` rules for this principal and intent. |
| 403 | `step_up_required` | Policy requires additional authentication before issuing the mandate. | Initiate the step-up flow via the coordinator. |
| 403 | `workspace_suspended` | The workspace has been suspended by an operator. | Contact the workspace operator. |
| 404 | `unknown_zone` | The `zone_id` in the request is not recognized. | Check `CARACAL_ZONE_ID` configuration. |
| 404 | `unknown_resource` | The requested resource is not registered in the workspace. | Register the resource binding via `caracal init resource`. |
| 429 | `quota_exceeded` | The principal has exceeded its mandate quota. | Wait for the quota window to reset or contact the workspace operator. |
| 500 | `internal_error` | Unexpected server error. | Check STS logs; file an issue if persistent. |

## Gateway errors

| HTTP | Error code | Meaning | Remediation |
|---|---|---|---|
| 400 | `missing_subject_token` | No `Authorization: Bearer` header on the request. | The SDK must attach the session bearer to all gateway calls. |
| 400 | `missing_resource_header` | No `X-Caracal-Resource` header on the request. | Set `X-Caracal-Resource: <rid>` in the SDK configuration. |
| 400 | `invalid_token_shape` | JWT is malformed or cannot be parsed. | Ensure the bearer is the mandate JWT from the coordinator session. |
| 401 | `mandate_expired` | Mandate has expired (past `exp` claim or within 35-second buffer). | Call `client.run()` again to request a new mandate. |
| 401 | `unknown_kid` | JWT `kid` not found in the zone's JWKS. | Check zone configuration; may indicate a key rotation in progress. |
| 401 | `invalid_signature` | JWT signature does not verify against the zone's JWKS. | Verify the mandate was issued by the correct STS instance. |
| 403 | `intent_mismatch` | Request path does not match `intent.resource` pattern in the mandate. | Ensure the `resource` pattern in the intent covers the actual URL. |
| 403 | `caveat_violated` | A caveat on the mandate was violated (wrong resource, wrong action, expired window). | Check which caveat fired; adjust the intent or policy constrain rules. |
| 403 | `replay_detected` | JTI seen more than once. This mandate has already been used. | Request a new mandate; do not reuse mandate JWTs. |
| 403 | `revoked` | Session has been revoked; mandate is no longer valid. | Do not retry; the session was terminated by an operator or the delegating principal. |
| 403 | `caller_id_injection` | Inbound request contains `X-Caracal-Client-ID` header. | Remove the header from the client; only the gateway sets it. |
| 403 | `path_traversal` | Request path contains `..` segments. | Fix the request path. |
| 502 | `upstream_error` | The upstream returned a non-success response. | Check the upstream service; the gateway forwards upstream error bodies unchanged. |
| 503 | `sts_exchange_failed` | Gateway could not reach STS for token exchange. | Check STS health (`GET /healthz`). |

## Coordinator errors

| HTTP | Error code | Meaning | Remediation |
|---|---|---|---|
| 400 | `invalid_delegation_scope` | Delegation scope exceeds the delegator's current mandate scope. | Narrow the delegation scope to a subset of the delegator's authorized scope. |
| 400 | `delegation_depth_exceeded` | Maximum delegation chain depth reached. | Reduce the depth of the agent delegation chain; default max is 3. |
| 401 | `invalid_delegation_token` | Delegation token is invalid, expired, or not recognized. | Issue a new delegation from the orchestrator. |
| 404 | `session_not_found` | Session ID does not exist or has expired. | Re-establish the session via `client.run()`. |
| 409 | `session_already_bound` | Identity already bound to this session. | Each session can only have one bound identity. |

## SDK error mapping

| SDK exception | Underlying HTTP error |
|---|---|
| `CaracalError.MandateExpired` | Gateway `mandate_expired` (401) |
| `CaracalError.PolicyDenied` | STS `policy_denied` (403) |
| `CaracalError.Revoked` | Gateway `revoked` (403) |
| `CaracalError.IntentMismatch` | Gateway `intent_mismatch` (403) |
| `CaracalError.ReplayDetected` | Gateway `replay_detected` (403) |
| `CaracalError.Unauthenticated` | STS `unauthenticated` (401) |
| `CaracalError.SessionNotFound` | Coordinator `session_not_found` (404) |
