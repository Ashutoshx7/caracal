- Scope: `playbooks/mapping/` contains playbooks for translating external provider and resource dashboards into Caracal fields.
- Must focus on provider-to-Caracal and resource-to-Caracal field mapping.
- Must help agents identify issuer, audience, callback URL, client ID, resource identifier, scopes, claims, tenant, and environment values.
- Must explain which provider values become Caracal applications, resources, scopes, grants, and policy inputs.
- Must tell agents to use screenshots or copied dashboard labels only to identify field meanings.
- Must use placeholders for all provider credentials, tenant IDs, domains, app IDs, client secrets, and private keys.
- Must tell agents to ask for redacted values or local environment variable names instead of raw secrets.
- Must not instruct users to paste provider secrets, access tokens, refresh tokens, private keys, or screenshots containing secrets into an AI chat.
- Must not place SDK implementation steps or Rego policy authoring as the main content of this layer.
- Must not assume every provider uses the same naming for resources, scopes, audiences, or claims.
